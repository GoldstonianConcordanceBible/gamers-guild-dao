# Repository and Documentation Links

GGUILD becomes stronger when its documentation and repositories are linked clearly.

## Core Documentation Objects

The main documentation objects include:

- GitBook homepage
- core docs
- whitepaper
- live system pages
- university pages
- doctrine pages
- roadmap pages
- community and operations pages

## Core Repository Objects

The main repository objects include:

- gamers-guild-dao repository
- GitBook folder
- contracts layer
- agents layer
- frontend and backend components
- university integration materials

## Why This Matters

Documentation should not exist in isolation.

Repositories should not exist in isolation.

Each layer should reinforce the others.

## Structural Principle

GitBook explains.

GitHub implements.

The knowledge graph connects.

## Final Point

The better the links between repositories and documentation, the easier it becomes to preserve continuity across the entire ecosystem.

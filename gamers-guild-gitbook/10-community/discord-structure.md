# Discord Structure

Discord should be organized so the community is easy to understand.

## Why This Matters

A messy Discord makes a project feel fake or abandoned.

A clean Discord makes the project feel alive and structured.

## Suggested Channel Groups

### Start Here
- welcome
- start-here
- announcements

### Community
- general
- introductions
- questions

### Governance
- proposals
- governance-discussion

### Build
- content
- contributors
- ideas

### Progress
- roadmap-updates
- kpi-updates

### Support
- help
- resources

## Principle

Every channel should have a purpose.

If a channel has no purpose, it becomes noise.

## Goal

The Discord should reflect the same structure and seriousness as the GitBook.

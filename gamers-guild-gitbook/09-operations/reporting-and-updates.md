# Reporting and Updates

A serious project should show progress regularly.

## Why Reporting Matters

Reporting builds:

- trust
- visibility
- accountability
- momentum

## What To Report

GGUILD can report on:

- roadmap progress
- community growth
- contributor activity
- documentation updates
- governance actions
- system improvements

## Good Update Types

Useful updates include:

- weekly summaries
- KPI snapshots
- contributor spotlights
- milestone announcements
- roadmap movement

## Best Practice

Updates should be:

- short
- clear
- repeatable
- honest

## Rule

Do not only post when something big happens.

Regular reporting is part of what makes the project feel real.

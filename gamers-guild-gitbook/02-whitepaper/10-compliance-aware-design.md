# Compliance-Aware Design

GGUILD is being documented with compliance awareness and message discipline.

That does not mean claiming approval or certainty.

It means describing the project responsibly.

## Core Principle

The project should be described through:

- utility
- participation
- governance
- contribution
- ecosystem function

It should not be described through promises of profit or guaranteed upside.

## Safe Framing

GGUILD is best described as:

- a utility-oriented token
- a governance-linked token
- a participation-focused ecosystem
- a gaming-native Sub-DAO case study

## What To Avoid

The project should avoid language that suggests:

- guaranteed return
- passive profit
- risk-free participation
- government approval
- regulatory clearance
- certain appreciation

## Why This Matters

Clear messaging helps reduce confusion and strengthens credibility.

It also helps keep the project focused on what it is actually trying to build.

## Important Note

Documentation alone does not create compliance.

Public behavior, public messaging, launch structure, and legal review still matter.

## Final Point

Compliance-aware design means speaking carefully, clearly, and honestly about what GGUILD is and what it is not.

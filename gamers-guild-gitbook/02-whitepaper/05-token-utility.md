# Token Utility

GGUILD is designed to function as a utility-oriented and governance-linked token.

Its value inside the ecosystem should come from what it allows the community to do, not from vague promises.

## Utility Categories

GGUILD may be used for:

- contributor rewards
- governance participation
- creator incentives
- community access
- ecosystem coordination
- participation recognition

## Reward Utility

GGUILD can be used to recognize and reward actions that help the ecosystem grow, such as:

- community support
- content creation
- education
- gaming-related contribution
- system-building efforts

## Governance Utility

GGUILD can support participation in:

- proposal discussions
- ecosystem direction
- community priorities
- governance votes

## Access Utility

GGUILD may also support access to:

- special community areas
- events
- selected ecosystem features
- contributor opportunities

## Design Principle

A token becomes stronger when its utility is:

- clear
- documented
- understandable
- tied to real actions
- connected to community value

## Final Point

GGUILD is intended to support participation and coordination inside a structured gaming ecosystem, not exist only as a speculative object.
